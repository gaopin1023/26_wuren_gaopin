完成这个作业我是建立了一个anoconda的隔离环境，叫做cone\_env，然后用的jupyter notebook写的代码

代码流程及细节：我将训练和测试代码分开了，感觉这样更清晰一点

训练代码及细节：1.先导入需要的库和工具：torch（深度学习核心框架）、torch.nn（网络层、损失函数定义）、torch.optim（优化器）、torch.utils.data.DataLoader（数据批量加载器）、torchvision.transforms（图像预处理工具）。

2\.再进行数据预处理，统一图片格式和大小，此处做了数据增强，将图片随机反转避免过拟合，并运用ToTensor()将 PIL 图像转为 PyTorch 张量，同时把像素值从 \[0,255] 缩放到 \[0,1]；还用了Normalize()，通过均值、方差标准化，将像素值映射至 \[-1,1]，加速网络收敛、提升训练稳定性；

3\.加载数据集：dataset/train,同时加载test1作为训练时的测试集

4\.搭建网络结构模型：4卷积 + 4池化 +2全连接以及激活函数ReLU，给网络引入非线性，用的是Conv2d、MaxPool2d、Linear，然后定义前向传播函数，定义数据流动流程为卷积 → 激活函数 → 最大池化，特征提取完成后将多维特征展平为一维（全连接层只能接收一维），然后送入全连接层完成分类（补充网络维度计算：模型输入为3通道、128×128 的 RGB 图像，每一组池化层尺寸缩小为原来的 1/2，经过 4 次池化后，最终特征图尺寸为 128/(2\*4) = 8×8；第四层卷积输出通道数为128，因此展平后特征维度为：128 × 8 × 8 = 8192。

5\.配置训练基础参数，如GPU还是CPU，训练轮数，优化器和损失函数等，此处的优化器学习率一开始为0.001，因过早达到最大准确率，故改为0.0005，此时10轮下来最后的准确率较为可观（此处补充对于优化器和损失函数的理解：损失函数指出预测和真实的差距，即告诉我错多少，在前向传播之后计算；梯度在计算完损失之后计算，即反向传播，告诉模型往哪个方向改，能够让损失变小，预测更准确，梯度方向是损失变大的方向，梯度下降则可减小损失；优化器根据梯度，负责改模型参数（卷积、全连接层权重），而学习率则控制参数修改的幅度，优化器更新幅度=梯度\*学习率）

6\.开始训练并打印结果：此处注意每次都要梯度清零，因为调参之后梯度会变，PyTorch 默认会在批次间累加梯度，不清零会导致梯度计算错误。同时测试的时候不要算梯度，用来提高训练速度

7\.最后记得保存模型，该文件仅存储网络的权重参数，不包含网络结构，这也是后续测试代码需要重新搭建相同网络的原因。

网络模型设计（前面已提到，此处再提一下）

整体架构：采用轻量化卷积神经网络，也许算是一个创新点吧。主干为 4 组卷积 + 激活 + 最大池化的特征提取模块，尾部使用 2 层全连接层作为分类头，结构简洁、参数量小，训练成本较低

现有局限：网络未加入批量归一化、Dropout 等正则化层，面对场景差异较大的陌生样本时，泛化能力仍有提升空间。



然后总结一下创新点：

创新点 1：调试学习率，解决模型过早收敛问题我在训练中发现使用一般学习率0.001时模型准确率快速达到最大，因此将学习率从0.001下调至0.0005，放缓参数更新步长，提升模型整体稳定性。

创新点 2：训练代码与测试代码分离,训练代码负责网络训练、反向传播、保存权重；测试代码负责加载权重、前向传播、统计准确率。两者完全隔离，互不干扰，代码逻辑清晰。

创新点 3：引入随机水平翻转数据增强，低成本抑制过拟合，避免模型单纯死记训练集图片特征，提升模型对不同拍摄角度锥桶的适应能力，缓解小数据集下容易出现的过拟合问题。

创新点4（ maybe)：采用轻量化网络结构，训练成本低。
