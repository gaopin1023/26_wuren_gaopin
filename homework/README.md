在这个代码里面我发现了一个有点瑕疵的点，就是代码里面只比较了f的大小，如果f相同的话就取最先出现的f，我感觉是不太好的，这样容易局部最优或者找到的可能不是最优，在面对复杂问题里面出错的可能性更大，可以改进为f相等的话就比较h, h相等的话，才取最先出现的f。



作业2：我发现线性回归与岭回归模型在房价数据集上的表现，与数据预处理、模型超参数选择和正则化强度有很大关系，整体测试集决定系数R²稳定在0.6-0.7之间。

在数据处理阶段，我一开始因为没有彻底删除文本列，导致矩阵运算报错，后来通过‘select\_dtypes(include=\[np.number])’只保留数值列，并配合删除缺失值，才解决了类型错误的问题。

无正则线性回归的模型训练过程比较简单，直接通过正规方程求解，训练集误差较低，但存在轻微过拟合现象。岭回归通过引入L2正则项，对权重进行惩罚，有效缓解了过拟合问题，测试集表现更稳定。

在调整超参数alpha的过程中，我发现alpha值的大小对模型性能影响明显：alpha过小时惩罚不足，模型和普通线性回归差异不大；alpha过大时惩罚过度，模型出现欠拟合。经过多组对比测试，alpha=0.1时模型的泛化能力表现最好。

我在写这个作业的核心难点在于手写正规方程的矩阵运算，尤其是构造带偏置项的特征矩阵和处理数据类型错误，花了不少时间调试。
